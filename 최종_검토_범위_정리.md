# 최종 검토 범위 정리

작성일: 2026-07-10

이 메모는 다음 검토 때 같은 범위를 반복하지 않기 위한 기록이다. 아래 항목은 이미 다른 채팅방 작업과 이번 최종 배포 검토를 합쳐 확인한 범위다.

## 이미 반영된 작업

- 배포 안정화
  - PC, 태블릿, 일반 모바일, 카톡 브라우저, 320px 작은 화면까지 확인했다.
  - 340px 이하 작은 폰은 사진 카드가 1열로 표시되도록 보정했다.
  - 앱 주소에 `board=`가 남지 않도록 처리했다. 특정 대지를 열어도 주소창 복사 시 기본 목록 링크가 되도록 정리했다.
  - 외부 스크립트 로딩이 지연되면 10초 안에 실패 처리하고 화면은 로컬 저장 모드로 열리게 했다.
  - favicon 404 오류를 제거했다.

- 사진 등록 및 출력
  - 여러 장 첨부 후 완료 안내가 뜨도록 수정했다.
  - 기존 사진대지 출력 미리보기 3쪽 생성과 사진 로드 실패 0건을 확인했다.
  - 카톡 브라우저 PDF 저장 제한은 안내 문구로 대응한다.
  - 사진대지 출력 제목 위 여백은 30mm로 조정했다. 표도 같이 내려가며 아래 여백은 약 32.2mm다.

- 관리자 모드 및 온도측정
  - 일반 사용자에게 온도측정 탭은 숨김/비활성 상태다.
  - 관리자 모드에서 온도측정은 기본 `측정 1`, `측정 2`만 표시된다.
  - 관리자 모드에서 온도측정 칸 추가/삭제가 가능하다.
  - 습윤양생 일차 삭제와 온도측정 칸 삭제는 서로 사진을 지우지 않도록 분리했다.

- 사진 시간 기준
  - 조작방지 기준은 사진 EXIF가 아니라 앱에 등록된 순간의 `검증일시`다.
  - 관리자 화면에는 `검증 ... · 자동 압축` 기준으로 표시된다.
  - 카톡 전송, 갤러리 편집, EXIF 변경 여부와 관계없이 공식 기준은 앱 등록 시각이다.

- 기준 문서
  - KCS/EXCS 기준 패널과 전문 이미지 보기 정상 확인했다.
  - KCS 전문은 11쪽, 첫 이미지 로드 정상 확인했다.

## 다음 검토 때 중복으로 보지 않아도 되는 범위

- 일반 모바일 레이아웃, 카톡 브라우저 레이아웃, 320px 작은 화면 레이아웃
- `board=` 주소 공유 위험
- 여러 장 첨부 완료 안내
- 기준 전문 보기 기본 동작
- 출력 제목 위 여백 30mm 적용 여부
- 관리자 모드 온도측정 기본 2칸 구조
- 관리자 메타 표시가 `검증일시` 기준인지 여부

## 다음에 보면 좋은 범위

- 실제 배포 URL에서 Supabase 저장/조회/업로드가 현장 휴대폰들에서 동시에 되는지
- 카톡 인앱 브라우저에서 PDF 저장 실패 시 안내가 충분히 전달되는지
- 전국 현장 확장 시 현장별 분리, 권한, 저장 용량, 삭제 정책
- 기준 문서 개정 시 `standards` 자료와 검색 인덱스 갱신 절차

## 운영상 주의

- 인터넷이 끊기면 서버 저장소에 올릴 수 없으므로 해당 브라우저 안에만 임시 저장된다. 연결 복구 후 새로고침/재등록 확인이 필요하다.
- 전체 배포는 `?board=`가 없는 기본 링크로 한다. 현재 앱은 `board=`가 들어와도 주소창에서 자동 제거한다.
- Git 저장소가 아닌 일반 폴더 상태라 변경 이력은 이 메모와 파일 날짜 기준으로 관리한다.
